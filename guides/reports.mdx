## Report Types

<CardGroup cols={2}>
  <Card title="Executive Summary" icon="briefcase">
    High-level overview for leadership and non-technical stakeholders.
  </Card>
  <Card title="Technical Report" icon="code">
    Detailed findings with proof-of-concept exploits for development teams.
  </Card>
  <Card title="Compliance Report" icon="shield-check">
    Findings mapped to compliance frameworks (OWASP, PCI-DSS, SOC2).
  </Card>
  <Card title="Delta Report" icon="arrows-left-right">
    Changes since last assessment—new, fixed, and recurring issues.
  </Card>
</CardGroup>

## Generating Reports

<Steps>
  <Step title="Navigate to Reports">
    Go to **Reports** → **Generate New Report**.
  </Step>
  <Step title="Select Target(s)">
    Choose which targets to include in the report.
  </Step>
  <Step title="Choose Report Type">
    Select Executive Summary, Technical, or Compliance format.
  </Step>
  <Step title="Configure Options">
    - Date range
    - Severity filter
    - Include/exclude sections
    - Branding options
  </Step>
  <Step title="Generate & Download">
    Click **Generate Report** and download as PDF.
  </Step>
</Steps>

## API Access

Generate reports programmatically:

<CodeGroup>
```bash cURL
curl -X POST 'https://api.indiesecurity.com/v1/reports' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "target_ids": ["target_123"],
    "type": "technical",
    "format": "pdf",
    "date_range": {
      "from": "2024-01-01",
      "to": "2024-01-31"
    }
  }'
```

```python Python
import requests

response = requests.post(
    'https://api.indiesecurity.com/v1/reports',
    headers={'Authorization': 'Bearer YOUR_API_KEY'},
    json={
        'target_ids': ['target_123'],
        'type': 'technical',
        'format': 'pdf'
    }
)

report_url = response.json()['download_url']
```
</CodeGroup>

<Tip>
  Reports are retained for 90 days. Download important reports for long-term storage.
</Tip>

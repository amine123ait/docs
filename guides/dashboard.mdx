The Indie Security dashboard provides real-time visibility into your security posture across all targets.

## Dashboard Overview

<Frame>
  Your dashboard displays key metrics and active assessments at a glance.
</Frame>

## Key Metrics

<CardGroup cols={4}>
  <Card title="Total Findings" icon="bug">
    Count of all discovered vulnerabilities across targets.
  </Card>
  <Card title="Critical Issues" icon="triangle-exclamation">
    High-priority vulnerabilities requiring immediate attention.
  </Card>
  <Card title="Active Scans" icon="spinner">
    Currently running assessments.
  </Card>
  <Card title="Coverage" icon="chart-pie">
    Percentage of application surface tested.
  </Card>
</CardGroup>

## Sections

<Tabs>
  <Tab title="Overview">
    The main dashboard showing:
    - Security posture score
    - Vulnerability trend charts
    - Recent findings
    - Active assessments status
  </Tab>
  <Tab title="Targets">

    - Add/edit target configurations
    - Configure authentication
    - Schedule recurring assessments

  </Tab>
  <Tab title="Vulnerabilities">

    - PDF export
    - Executive summaries
    - Technical details
    - Compliance mapping
  </Tab>

## Target Management

<Steps>
  <Step title="Add New Target">
    Click **+ Add Target** and enter:
    - Target name
    - Root domain/URL
    - Description
    - Environment type (Production/Staging/Development)
  </Step>
  <Step title="Configure Settings">
    Set up:
    - Authentication credentials
    - Scope restrictions
    - Excluded paths
    - Rate limiting preferences
  </Step>
  <Step title="Schedule Assessments">
    Configure recurring scans:
    - Daily/Weekly/Monthly frequency
    - Assessment mode (Auto/Vuln Testing/APT)
    - Notification preferences
  </Step>
</Steps>

## Real-Time Monitoring

During active assessments, monitor:

| Metric | Description |
|--------|-------------|
| Progress | Percentage of endpoints tested |
| Findings | Running count of discovered vulnerabilities |
| Coverage | Attack surface coverage percentage |
| Duration | Elapsed and estimated remaining time |

<Tip>
  Enable browser notifications to receive alerts when critical vulnerabilities are discovered.
</Tip>

## Filters and Search

Find specific data quickly:

```text
# Search examples
severity:critical target:production
type:xss status:open
discovered:last-7-days
```

## Notifications

Configure alerts for:

- New critical/high vulnerabilities
- Assessment completion
- Failed authentication
- Scheduled scan reminders

<Info>
  Notifications can be sent via email, Slack, or webhook integration.
</Info>
